1. Application Overview

This project is an AI food-ordering assistant where the user can discover restaurants, browse menus, add items, choose delivery/payment, place order, and track status through conversational UI.

It supports both:

  • Web chat (Next.js frontend + FastAPI backend)
  • Telegram bot channel

Core capability is tool-driven LLM orchestration with Zomato MCP tools, so the assistant does not hallucinate restaurants or menu data.

FastAPI Next.js LangGraph LangChain Tools MCP Gemini/OpenAI-compatible APIs Voice transcription Session state machine

2. Technology Stack and What I Learned

Backend Framework: FastAPI

  • Created typed request/response models with Pydantic.
  • Built async endpoints for chat, voice upload, order status polling, session reset, and QR image serving.
  • Learned to separate route layer from orchestration layer for maintainability.

Agent Orchestration: LangGraph + LangChain Tools

  • Implemented ReAct-style graph with tool calling from model context.
  • Built deterministic guards for critical flows (address resolution, cart creation, order confirmation).
  • Learned that hybrid architecture (deterministic logic + LLM reasoning) is more reliable than pure prompting.

MCP Integration

  • Connected to Zomato MCP server via stdio client session.
  • Added retries, timeout handling, cooldowns, and recoverable reconnect logic.
  • Learned how MCP decouples business tools from LLM provider choice.

Voice Pipeline

  • Added browser voice recording (MediaRecorder) in frontend.
  • Added backend voice upload endpoint and transcription service.
  • Implemented provider-compatible audio transcription paths (Gemini and OpenAI-compatible).

Frontend: Next.js

  • Built proxy routes to avoid CORS and keep backend URLs server-side.
  • Integrated context-aware quick replies, status polling, and inline QR rendering in chat.
  • Learned Next.js 16 dynamic route param changes (params must be awaited in route handlers).

System Design Learnings

  • Session-driven conversation state is essential for multi-turn ordering.
  • Tool output normalization and fallback search prevents false "no result" failures.
  • Telemetry logs are critical for debugging real-world voice/order issues.

3. End-to-End Ordering Flow (Start to Finish)

High-Level User Journey

  1. User opens chat and asks for cuisine/restaurant.
  2. Assistant ensures delivery location/address context exists.
  3. Assistant fetches restaurants via MCP tool.
  4. User selects restaurant, assistant fetches menu.
  5. User adds items (text or voice), assistant resolves quantities + variants.
  6. Assistant shows cart and payment choices.
  7. Assistant confirms order, creates cart, checks out, renders QR if UPI.
  8. User pays and asks to track order.
  9. Assistant tracks current order and provides status updates.
Web/Telegram Input -> FastAPI /api/chat/message or /api/chat/voice-upload -> LangGraphFoodAgent.process_message -> Deterministic checks (address, state, parsing, tracking command) -> MCP tool call(s) through ZomatoMCP client -> Structured response + session updates -> Frontend renders bot text, quick replies, cart, QR, order polling

Addressing "Order for another person"

During active checkout, delivery context can now be updated without destroying cart/menu state. This allows ordering to a different address/person while preserving the current flow.

4. How LLM Works with MCP in My Backend

Core Principle

The LLM is used for reasoning and language understanding, but external data/actions always come from MCP tools. This avoids hallucination and keeps actions auditable.

Execution Model

  • System prompt enforces guardrails: no fabricated data, short actionable responses, confirm before order placement.
  • LangGraph builds tool-enabled agent graph with discovered MCP tools.
  • For each message, deterministic handlers run first for sensitive flows (tracking, address updates, checkout states).
  • If deterministic path is not enough, graph invocation executes tool-augmented reasoning.

Reliability Controls

  • Multi-model fallback list (if one model fails due to limits/unavailability).
  • Provider error classification with user-friendly fallback text.
  • MCP timeout, reconnect, and recoverable retry behavior.
  • Address ID refresh when tool returns invalid address.
Interview line: "I used LLM for intent + language, but business correctness is guaranteed by deterministic checks and MCP tool outputs."

5. Voice + MCP + LangGraph Flow

Voice Input Path

  1. Frontend records audio via MediaRecorder.
  2. Frontend posts multipart data to Next.js proxy route.
  3. Proxy forwards to backend /api/chat/voice-upload.
  4. VoiceService transcribes audio to text.
  5. Transcribed text enters same agent flow as text chat.
  6. Response returns with thinking steps + state/cart updates.

Voice Failure Handling

  • Graceful user-facing message on failures.
  • Telemetry events for timeout/rejection/upload failure.
  • Fallback path supports both Gemini-native and OpenAI-compatible transcription APIs.

7. Backend Modules - Component-by-Component Flow

main.py (App Bootstrap)

  • Loads env, starts FastAPI, enables CORS.
  • Boots Telegram bot in lifespan task.
  • Conditionally enables web chat routes.

routes/chat.py (HTTP Contract)

  • Text endpoint: /api/chat/message
  • Voice endpoints: /api/chat/voice-message and /api/chat/voice-upload
  • Session endpoint: /api/chat/session/{user_id}
  • Tracking endpoint: /api/chat/order-status/{user_id}
  • QR endpoint: /api/chat/qr/{file_name}

services/langgraph_agent.py (Brain)

  • Deterministic intent handlers for tracking, address selection, payment/confirmation.
  • Menu parsing, item matching, variant handling, cart building.
  • Tool schema adaptation and agent graph invocation.
  • Search fallback logic and rating-based filtering.

services/zomato_mcp.py (Tool Transport)

  • MCP session connect/list/call operations.
  • Retry/reconnect handling for transient transport failures.
  • Timeout governance for tool calls.

services/voice_service.py (Speech-to-Text)

  • Gemini-native transcription path.
  • OpenAI-compatible audio path as fallback.
  • MIME normalization and transcript extraction.

services/session_service.py (Conversation Memory)

  • Persists sessions in JSON for restart continuity.
  • Maintains state machine, cart, selected restaurant, current order ID.
  • Supports safe delivery-context updates for "order for another person" scenarios.

10. Detailed Ordering Flow (Search to Payment)

Demo Script You Can Read in Video

  1. User says: "Find nearest pizza restaurants".
  2. Assistant checks if location/address is available in session.
  3. If location is missing, assistant asks for city/area or saved address selection.
  4. Assistant resolves address_id using MCP saved-address tool.
  5. Assistant calls restaurant discovery tool with keyword and address_id.
  6. Assistant returns ranked restaurants with rating + delivery ETA.
  7. User selects restaurant by number (for example: 1).
  8. Assistant calls menu tools and maps item numbers to menu metadata.
  9. User adds items by text or voice (for example: "2 Margherita and 1 Corn and Cheese").
  10. Assistant parses quantities, matches item names, resolves variant IDs.
  11. Assistant updates cart and shows subtotal + payment options.
  12. User selects payment type: UPI QR or COD.
  13. Assistant shows final order summary and asks for confirmation.
  14. User replies yes.
  15. Assistant creates Zomato cart, checks out, and returns QR in chat for UPI.
  16. User completes payment and asks: "track my order".
  17. Assistant tracks current order ID and returns normalized, clean status updates.

Backend Path for Same Flow

Frontend voice/text -> /api/zomato/chat or /api/zomato/voice-upload (Next.js proxy) -> FastAPI /api/chat/message or /api/chat/voice-upload -> LangGraphFoodAgent.process_message -> SessionService state read/update -> ZomatoMCP.call_tool(...) -> Response composer (cart/state/restaurant/order) -> Frontend renderer (message, quick replies, cart, QR, polling)

Tool/API Mapping by Stage

Stage Primary Calls Output to User

Restaurant search get_restaurants_for_keyword Top restaurants with rating and ETA
